Für Ungeduldige:

Man kann sich schon auf dem Server registrieren oder einloggen.

Bekannte Probleme beim Einloggen:

  1. gelöst: 3D-Fibs Registrierung eines neuen user sollte jetzt funktionieren.
  2. telnet wird beim Login noch nicht korrekt unterstützt.
Bekannte Probleme beim Spielen:
  1. Man braucht einen Partner.
  2. in Arbeit: Das Spiel kann hängen bleiben, wenn der Server die Zahl der möglichen Züge falsch berechnet. Korrekt funktioniert es bisher nur beim Einwürfeln von der Bar und natürlich in allen Fällen, wo man 2 Züge (4 beim Pasch) machen kann.
  3. in Arbeit: Es gibt noch ein paar Ungereimtheiten beim Doppeln, Resign und Abrechnen.
  4. Es gibt noch kein resume.
Wenn man 'rausfliegt'......dann ist ein Fehler aufgetreten :) ....... einfach gleich wieder einloggen.

Über TigerGammon

Was ist TigerGammon?

TigerGammon ist ein freier Backgammon Server ganz in der Art von FIBS. Mit TigerGammon soll die Institution FIBS am Leben erhalten und erneuert werden.
TigerGammon bietet die Möglichkeiten und die Community wie FIBS. Im Laufe der Zeit kommen Funktionen hinzu, die über das hinausgehen, was auf FIBS möglich ist. Dazu gehören
  • Chouette
  • MoneyGames
  • ....

Wer macht TigerGammon?

Andreas Hausmann macht TigerGammon. Er ist ein Fibster, der mit der Leistung von FIBS ebenso unzufrieden ist, wie viele andere. Er hat verschiedentlich Hilfe angeboten, hat aber jetzt einen neuen Server gebaut, der das kann, was FIBS leistet. Und mehr.....
Zu erreichen ist er unter andreas ( a t ) tigergammon.com.
Oder versuch's auf Skype bei tigerbackgammon.

Was ist mit FIBS?

FIBS erleidet zunehmend den Kollaps. Spielen dort mehr als 200 Spieler, dann steigen die Antwortzeiten des Servers in's Unerträgliche. Der dynamische Timeout, der FIBS davor schützen soll, von zu vielen untätigen Spielern blockiert zu werden, unterbricht Spiele, macht es schwer, ein Spiel nur zu beobachten, und nervt die alten und jungen Hasen, die gerne nur herumhängen und mit ihren Bekannten ein Schwätzchen halten wollen.

Was ist mit meinem FIBS Account?

Wenn du willst, holen wir den hierher.
Das heißt, dass du deinen FIBS-Nick und dein FIBS-Passwort eingibst. Dann werden auf FIBS die Daten ausgelesen und hier eingespeichert. Dieser Service ist noch nicht eingebaut. Aber nach der Testphase wird er zu Verfügung stehen.
Das heißt aber nicht, dass dein FIBS-Account stirbt. Es werden lediglich die Zahlen geholt. Wo du weiterspielst, entscheidest du.
Das Ganze steht unter dem Vorbehalt, dass der TiGa-Server nicht von FIBS gesperrt wird. n dem Fall kann auch TiGa nix mehr machen.

Was unterscheidet TigerGammon von FIBS?

FIBS hat technische Probleme, die daher rühren, dass die Software sehr alt ist.
TigerGammon ist neu. Endlich können neue Funktionen realisiert werden, auf die viele engagierte Spieler schon lange gewartet haben. Zu diesen Funktionen gehören:
  • Chouette
  • MoneyGames
  • TimeControl
  • ....
Aber auch einige Unannehmlichkeiten sollen verschwinden:
  • Kurze Timeouts
  • Lange Antwortzeiten, wenn mehr als 200 User eingeloggt sind
  • Dropper oder ungerechte Dropperbehandlung
  • Begrenzung auf 240 User
  • Sperren von IPs/Domains

Wie kann ich mich auf TigerGammon einloggen?

Der Server ist unter tigergammon.dyndns-home.com zu erreichen. In naher Zukunft wird das tigergammon.com sein, oder einfach ti-ga.com.

Also musst du im Moment in den "Server"-Einstellungen deines Clients tigergammon.dyndns-home.com eintragen.

Auf JavaFIBS ist es die Einstellung unter Preferences/Server. Dort trägt man als Host statt fibs.com einfach den neuen Server ein: tigergammon.dyndns-home.com. Der Port bleibt auf 4321.

Auf 3DFibs geht man unter Options/Preferences und landet auf der Registerkarte FIBS. Dort trägt man in das Feld Connection den Server ein: tigergammon.dyndns-home.com. Der Port bleibt auf 4321.

Denk dran. TigerGammon ist neu. Du hast dort vielleicht noch keinen Account und musst dich erst mal registrieren.

Ist mein Passwort sicher?

Im Moment ist das Passwort in Klarschrift abgelegt. Das ist genau wie bei FIBS. Das Passwort ist also genau so sicher, wie bei FIBS.

In den Server-Protokollen sind die Passwörter nicht zu lesen. Dennoch gibt es für den Betreiber prinzipiell die Möglichkeit, Passwörter aus der Datenbank auszulesen. Während der Testphase könnte es sein, dass das im Verlauf einer Fehlersuche geschieht.

Lange Rede, kurzer Sinn. Der Betreiber will dein Passwort nicht wissen. Im Verlauf der Tests könnte es passieren, dass er es liest. Auf Dauer sollte es so sein, dass die Clients die Passwörter verschlüsselt an den Server schicken. Dann wird alles gut.

Was geht noch nicht?

1) Es gehen noch nicht alle Meldungen zum Spiel. Meistens schreibt der Server Meldungen noch nicht korrekt ins Systemfenster, z.B. 'Starting a new game with user'. Das verwirrt den Client und er reagiert nicht immer wie gewünscht.
2) Daraus resultiert, dass beim Doppeln und Resign manche Sachen nicht genau funktionieren. Oftmals wird das Spiel korrekt gewertet, aber der Client stellt es falsch dar.
3) Der Server ermittelt noch nicht in allen Fällen die Zahl der möglichen Züge korrekt. Es kann zum Beispiel passieren dass man gegen Ende eines Spiels bei blockierten Sechsen mit einem 6er Pasch hängen bleibt. Der Server meldet dem Client, dass 4 Züge möglich sind, der Spieler kann sie aber nicht setzen.

Insgesamt sollte man noch nicht wettkampfmäßig spielen wollen :)

Folgende Features gehen noch nicht, sollen aber bald implementiert werden:
1) Speichern und 'resume' der gespeicherten Spiele.
2) Das Beobachten eines Spiels mit 'watch'.

Kommandos, die mit diesen beiden Punkten zu tun haben, z.B. 'show saved' oder 'whisper', gehen dementsprechend auch noch nicht.

Wie melde ich einen Fehler?

Das geht wirklich einfach. Man muss dem Server eine Zeile schreiben, in der das Wort "TODO" vorkommt. Es ist wichtig, dass alles groß geschrieben wird.
Es ist egal, ob man die Zeile im Chat schreibt. Man kann es auch als "command" schreiben. Dann kommt vielleicht eine Fehlermeldung, aber das ist egal. Die Hauptsache ist, dass der Server die Zeile mit "TODO" bekommt. Zum Beispiel: "TODO - der Cube geht nicht vom Board weg".
